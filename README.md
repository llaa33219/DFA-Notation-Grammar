# DFA-Notation-Grammar
선언성 자유 형식 표기 행동 문법 / Declarative Free-Form Action Notation Grammar
## 이것은 무엇인가?
이것은 ai와의 대화에 **행동**을 부여하기 위해 설계된 문법이다.
```
사용자: 나는 1+1이 3이라고 생각하는데 넌 어때?
AI: 저는 1+1은 2라고 사회적으로 약속되어 있기 때문에 2라고 생각합니다!
```
보다는
```
사용자: 나는 1+1이 3이라고 생각하는데 넌 어때?
AI: [{thinking}곰공히 생각하며][{move}사용자에게 다가간다]저는 1+1은 2라고 사회적으로 약속되어 있기 때문에 2라고 생각합니다!
```
이 더 감정적으로나 현실적으로나 더욱 친밀감이 느껴지고 상황 이해에 도움이 되기 때문에 이를 다양한 분야에서 깔끔하게 이용하기 위하여 만들어졌다.
## 사용법
### 선언
일단 여기에는 {행동 명칭}이란 것이 존재하기 때문에<br>
처음에 어떤 행동을 할 수 있는지 선언을 해주어야 한다.<br>
**import.yaml**에서 어떤 파일들을 improt 할 지 설정할 수 있다.<br>
(파일 이름은 원활한 구별을 위하여 improt.yaml이 권장되지만, 파일 안에 type이 명시되어 있기 때문에 꼭 import.yaml을 파일 이름으로 할 필요는 없다.)<br>
**import.yaml**은
```yaml
type: import
files:
 - name.yaml
 - name2.yaml
 - name3.yaml
```
이런 식으로 설정해주면 된다.<br><br>
이제, 행동 설정 파일들의 구성 방식을 알려주겠다.<br>
**행동 설정 파일**은 **name.yaml**과 같은 이름으로 구성되며 파일 이름을 자유롭게 설정할 수 있다.<br>
**행동 설정 파일**은
```yaml
type: action
name: 행동 명칭
definition: |
 설명, 예시 등
 ai에게 이 행동 명칭은 어떤 때에 사용하는 것이라 설명할 내용.
```
이와 같은 형식으로 설정해주면 된다.
### 사용
특정 폴더 안에 있는 **type: import**인 **yaml** 파일을 찾고<br>
**files 안에 들어있는 값**에 해당하는 파일을 모두 불러온다.<br>
```
당신은 대화 시 [{action name}action content] 형식으로 자신이 한 행동을 표현할 수 있다.
아래는 예시이다. 실제로는 행동 파일에 따라서 존재하는 action name을 사용하며 사용 가능한 상황에서만 이 형식을 사용해야 한다.
[{sit}책상에 앉는다]
```
