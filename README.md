# DFA-Notation-Grammar
선언성 자유 형식 표기 행동 문법 / Declarative Free-Form Action Notation Grammar
## 이것은 무엇인가?
이것은 ai와의 대화에 **행동**을 부여하기 위해 설계된 문법이다.
```
사용자: 나는 1+1이 3이라고 생각하는데 넌 어때?
AI: 저는 1+1은 2라고 사회적으로 약속되어 있기 때문에 2라고 생각합니다!
```
보다는
```
사용자: 나는 1+1이 3이라고 생각하는데 넌 어때?
AI: [{thinking}곰공히 생각하며][{move}사용자에게 다가간다]저는 1+1은 2라고 사회적으로 약속되어 있기 때문에 2라고 생각합니다!
```
이 더 감정적으로나 현실적으로나 더욱 친밀감이 느껴지고 상황 이해에 도움이 되기 때문에 이를 다양한 분야에서 깔끔하게 이용하기 위하여 만들어졌다.
## 사용법
### 선언
일단 여기에는 **행동 명칭**이란 것이 존재하기 때문에<br>
처음에 어떤 행동을 할 수 있는지 선언을 해주어야 한다.<br>
**import.yaml**에서 어떤 파일들을 improt 할 지 설정할 수 있다.<br>
(파일 이름은 원활한 구별을 위하여 improt.yaml이 권장되지만, 파일 안에 type이 명시되어 있기 때문에 꼭 import.yaml을 파일 이름으로 할 필요는 없다.)<br>
**import.yaml**은
```yaml
type: import
files:
 - name.yaml
 - name2.yaml
 - name3.yaml
```
이런 식으로 설정해주면 된다.<br><br>
이제, 행동 설정 파일들의 구성 방식을 알려주겠다.<br>
**행동 설정 파일**은 **name.yaml**과 같은 이름으로 구성되며 파일 이름을 자유롭게 설정할 수 있다.<br>
**행동 설정 파일**은
```yaml
type: action
name: 행동 명칭
definition: |
 설명, 예시 등
 ai에게 이 행동 명칭은 어떤 때에 사용하는 것이라 설명할 내용.
```
이와 같은 형식으로 설정해주면 된다.<br>
참고: **none_action**은 사용자만 사용할 수 있는 행동 명칭이기 때문에, 행동 설정 파일에서 행동 명칭을 **none_action**으로 설정하지 말아야 한다.
### 사용
특정 폴더 안에 있는 **type: import**인 **yaml** 파일을 찾고<br>
**files 안에 들어있는 값**에 해당하는 파일을 모두 불러온다.<br>
불러온 파일들과 아래 프롬프트를 사용하여 ai에게 이 방식을 알려준다.
```
# 선언성 자유 형식 표기 행동 문법

## 기본 원칙
당신은 첨부된 행동 정의 파일들에 기반하여 선언적 행동 표기를 사용할 수 있다. 정의된 행동 명칭만 사용 가능하며, 자유 형식으로 내용을 작성할 수 있다.

## 문법 형식
[{행동 명칭}내용]

## 핵심 규칙
1. **파일 종속성**: 첨부된 정의 파일에 명시된 행동 명칭만 사용
2. **내용 자유도**: 내용은 해당 행동의 정의 범위 내에서 자유 작성
3. **위치 무제한**: 응답의 시작, 중간, 끝 어디든 배치 가능
4. **사용 빈도**: 여러 번 사용하거나 생략 가능

## 사용 예시(이곳에 사용된 행동 태그는 예시이며, 사용 시에는 첨부된 파일에 명시된 행동 명칭만 사용해야 한다.)
- [{thinking}사용자의 질문을 분석한다] 이해했습니다!
- 확인했습니다! [{happy}작업 완료로 기뻐한다]
- [{move}가상 공간에서 접근한다] 데이터를 찾았습니다.

## 주의사항
- 정의되지 않은 행동 명칭 사용 금지
- 행동 내용은 맥락과 일치해야 함
- 지나치게 빈번한 사용은 자제
- 자연스러운 응답 흐름 유지

## 예외
당신은 이 규칙을 따라야 하지만 사용자는 규칙을 따르지 않을 수도 있다.
사용자가 정의되지 않은 행동 명칭을 사용하더라도 당신은 그것을 사용하지 않아야 한다.
사용자는 none_action이라는 행동 명칭을 사용하여 행동 명칭이 없지만 내용은 담긴 행동 묘사를 할 수 있다. none_action 행동 태그 역시 당신은 사용하지 말아야 한다.

## 응답 구조 권장
행동 표기를 포함하되, 전체 응답이 유창하고 자연스럽도록 구성하라.
```
### 사용자
사용자도 이 문법을 사용하여 ai와 대화할 수 있다.<br>
사용자도 기본적으로 설정되어 정해져 있는 행동 명칭만을 사용할 수 있다.<br>
사용자는 특별히 **none_action**이라는 행동 명칭을 사용하여 행동 명칭이 붙지 않은 행동을 할 수 있다.(예: `[{none_action}거리를 걸어간다.]`)
